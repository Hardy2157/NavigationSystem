# NavigationSystem 文档索引

为方便团队阅读与分发，原来的总说明文档已拆分为下面两份：

## 1. 架构文档

文件：`docs/architecture-guide.md`

适合先阅读这份文档的人：

- 第一次接触项目的新成员
- 需要先建立整体认知的人
- 主要关注系统分层、模块关系、调用链和运行流程的人

这份文档重点回答：

- 项目整体在做什么
- `app / gui / core` 三层如何协作
- 地图生成、路径规划、交通仿真、空间查询如何串起来
- 从一次用户操作到界面回显，中间经过哪些模块
- 阅读代码时应该先看哪里

## 2. 接口说明

文件：`docs/interface-reference.md`

适合优先阅读这份文档的人：

- 需要开始改代码的人
- 需要快速定位类职责和接口边界的人
- 准备扩展算法、GUI 或文件格式的人

这份文档重点回答：

- 核心类分别负责什么
- 关键字段和方法有哪些
- 谁依赖谁，谁调用谁
- 参数、返回值和重要语义是什么
- 新功能应该接到哪些接口上

## 3. 推荐阅读顺序

建议按下面顺序阅读：

1. 先看 `docs/architecture-guide.md`
2. 再看 `docs/interface-reference.md`
3. 最后按文档中的推荐顺序进入源码

## 4. 配套建议

如果你准备把文档发给组员，建议按下面的方式使用：

- 第 1 天：所有人统一阅读架构文档
- 第 2 天：按各自负责模块阅读接口说明
- 第 3 天：结合源码做一次 30 分钟的调用链复盘

这样效果会比直接把整仓库甩给组员更好。
